The main page of Cache displays two distinct areas: A Navigation Pane on the right and a Work Area on the left (with a beige backround).

At the top of the Navigation Pane is the Cache Logo image. Clicking on this image will always return you to the main page. Under the image is a menu bar with clickable buttons for 'Search', 'Add Item', 'Query', 'Help' and 'Quit'.

The area beneath the menu bar will vary depending upon the command being executed.

By default the Work Area displays the 10 most recently edited items.

The Work Area presents summary reports of items. Notable features include a link to Open the item if it is an electronic document as well as a link to View any associated images. The bottom line of the summary record has links to perform a Simple Search for associated Genres and Tags.

You can view a full report on the item by clicking on the small icon picture located to the left of the summary report.

Click on the 'Add Item' button in the menu bar. This will open a new, blank 'Adding New Item' page where you can input the attributes of the item. Note that many of the attribute sections are collapsed and hidden from view. Click on the attribute name to expand (or collapse) the section as needed.

When adding or editing an item the lower area of the Navigation Pane will display a list of previously used 'Attribute Pick Lists' (also expandable). Clicking on the attribute value in the Picklist will populate the corresponding field in the item edit form. Using the picklists encourages consist teminology and reduces the amount of typing you need to do. Note that this feature will replace any pre-existing data in the form field.

Remember to click the 'save' button at the top of the form. This will return you to the main page showing the most recently edited items. The newly edited item should appear at the top of the list.

You can click on the 'Cache' image at the top of Navigation Pane at any time to return to the main page without saving the item.

core

name (required): This is the short primary descriptor of the item.

authors: The author(s) of the book

description: A detailed description of the item.

id: This is a system assigned numeric identifier, it is not editable by the user.

genre* (highly suggested): A single broad grouping of items

tags: A list of related subject areas or concepts. Multiple values can be entered by separating the tags with commas.

location: Where is the item stored?

qty: The number of items covered by this record.

date completed: When did you finish reading this item?

publishing information

isbn: International Sellors Book Number

format: The format of the publication, hardback, softback, eboot, etc.

pages: How many pages?

edition: What edition?

language: What language is the item in?

publisher: Who is the publisher

published: When was it published?

images

icon: A small image to represent this item--should be 70 pixels wide and 100 pixels high for display on the item and search pages. Could be shared with related class of items. Icon images cannot be deleted in Cache because they may be used by multiple items. They can be manually deleted from the cache/application_images folder.

item image: A larger image of the specific item. Multiple images can be uploaded but not all at the same time. Upload one image, save the item, edit it again to upload the second image, etc. individual images can be deleted in this section by clicking the 'delete' box beneath the image and then saving the item.

valuation

cost: What did you pay to purchase the item?

value: How much would you want to sell this item?

acquired date: Date on which you acquired this item.

disposition: When did you get rid of this item?

links

web review/source: Supply an internet url for the item (for example to the seller's web site) or, if cataloging an electronic document you can load it into Cache here.

notes: Random notes about the item.

delete: To delete the item from Cache, click the 'enable' check box and then click on the 'delete' button. This will delete the record of the item but uploaded pictures, icons or electronic documents will remain in the Cache folders.

Click on the id number of the item in the search results page to edit an item record. You may need to perform a simple search for the item or use the 'Simple Search Attribute List' on the Navigation Pane in order to view the summary record of the item.

You must edit an item record in order to get to the delete function. Bring the item summary record up in the Work Area through a simple search. Click on the item icon to edit the item record. Once the item record is open in the Work Area then scroll to the bottom of the record. Expand the 'delete' category, check the 'enable delete' box and then click the 'delete item #' button.

Note that deleting an item removes the item record from Cache but it does not automatically remove associated images or uploaded documents. Those remain in the Cache resource folders and can be removed manually.

The first and easiest way to execute a 'Simple Search' is by clicking on one of the attribute values in the 'Simple Search Attribute List' in the Navigation Pane. Every item containing the value in the attribute field will be displayed in the Work Area. The simple search lists are automatically generated from the values entered for individual attributes in item records.

The second way to perform a simple search is to use the search function at the top of the Navigation Pane beneath the Command Menu. Enter a search term in the text box, select whether you want a broad ('containing') or an exact search and select the field you wish to search. Finally, click on the 'search' button. All items which match your search term will be displayed in the Work Area sorted alphabetically by name.

A broad search (e.g., 'containing') for 'WWI' will return both 'WWI' and 'WWII' because both values contain the sequence 'WWI'. An exact search for 'WWI' will return items with only 'WWI'.

You can search for multiple terms in a single field by separating the terms with tildes '~'.For example, searching for 'ships~boats' contained in the description field will return all items with 'ships' OR 'boats' in the description. However, if you perform an 'exact' search for 'ships~boats' Cache will look for the exact term 'ships~boats' including the tilde! 'Exact' searches do not work with multiple search terms.

Unlike simple searches, Queries allow more flexibility in defining search parameters and formatting the results. The Query page is accessible by clicking the 'Query' button of the Command Bar on the Navigation Pane.

The Query page displays every attribute field and the user can select which attributes to display in the Query results by giving them an order in which to display. Query results can be filtered by entering values in the attribute text boxes and, if desired, selecting whether the field matches the term exactly.

By default, queries filtered by multiple fields are boolean 'AND' searches--field 1 must contain 'THIS' AND field 2 must contain 'THAT'. However, a boolean 'OR' search can be conducted by preceeding the second or subsequent search terms with a tilde '~'. In other words, field 1 contains 'THIS' OR field 2 contains '~THAT'

Likewise, a single field can be searched for multiple terms. For example, querying field 1 for 'THIS~THAT~THOSE' would return records with 'THIS' OR 'THAT' OR 'THOSE' in field 1, however, remember the limitation of 'exact' searches combined with '~'. They do not work!

Lastly, you can sort the query results by any field. Once the query results have been displayed, you can reorder the data in either ascending or descending order by clicking on the column heading. Note that this reordering will override the order assigned when selecting the query fields.

A summary row will be provided for each value in the first ordered field. For example, if 'location' is the first field then a summary row will be provided for each unique location.

Whenever the 'id' field is included in a query the resulting report will display the 'id' as a link to the item edit page.

Providing the query a name will add it to the Saved Query list. Clicking on the Saved Query name will recall the query parameters for repeated querying. Unamed queries are lost once the query page is exited.

Query results open in a new browser tab. You can simply close the results tab to return to the query page, modify it if needed and run it again.

You'll probably need to play around with the query parameters to get a better understanding of how they function.

The Query page gives you a number of options to create graphical charts of your collection. Note that you will probably want to exclude the column one summary when creating charts.

  • standard: just the report, no chart created.
  • bulk update: provides a form with a list of records from the report where you can update multiple records at a time.
  • chart - pie: creates a pie chart using the first ordered field of the query.
  • chart - bar: creates a bar chart using the first ordered field of the query.
  • chart - bar-stacked: creates a stacked bar chart using the first ordered field for the x axis and the second for the y axis.
  • chart - bar-grouped: creates a bar chart using the first ordered field and groups the bars by the second ordered field.
  • chart - timeline: creates a scatter chart showing the acquisition and completion dates of items in the report.
  • drawer labels: produces a report formatted to create 8.5x11in labels showing items and locations.
  • tab delimited: produces a report without formatting for copying into a spreadsheet.

Cache stores data in simple text files in YAML format in the 'cache/' folder. The item records are saved in 'application_items.yml'. Saved reports are saved in 'application_reports.yml'. To back up your data simply copy these files to wherever you keep your backup data. To restore your data simply copy your backup *.yml files to the cache folder.

YAML files can be viewed with any text editor, though it's not recommended to change the files unless you are familiar with the YAML data format.

If you upload images they will be stored in the cache/application_images/ folder. If you upload electronic documents they will be saved in the cache/application_publications/ folder. Remember, if you delete an item in Cache, associated uploaded images and documents are NOT removed from these folders. You'll need to remove them manually.

Technical Requirements

Cache is written in the Ruby programming language. Ruby must be installed and configured on the computer for Cache to run. Details on installing Ruby can be found at Ruby-lang.org. Cache uses the Webrick web server which up until Ruby version 3.0 was part of the Ruby core distribution. With 3.0, the Webrick Gem must be specifically installed. Cache also depends upon the YAML gem, though at the time of this writing that is still part of the Ruby Core Distribution.

Web Server? What's this about a web server?

Yes, Cache relies on the Webrick web server running on your local machine and you access it via your preferred web browser. Given the right information, other computers can access the server. That's what servers do. The Cache code does not do a significant amount of input validation, therefore, it is recommended that you keep your computer secure behind a firewall when running Cache.

Quitting Cache

Notice the quit button in the upper right hand corner of Cache. Clicking this button will shut down Cache and the Webrick web server. The quit button will not close your browser. However, if you simply shut down your browser without clicking the quit button, both Cache and Webrick will continue to run in the background and you may not even realize it unless you look at the process list of your operating system. As long as Cache is running in the backgroud, you will not be able to start another instance of it and, depending upon how you try start it, it may not give you any indication as to why. If Cache is running in the background you can get to it by opening up your browser and going to localhost:8000/items.

So how do you quit Cache after you've closed your browser? Simply open your browser again and type localhost:8000/quit into the address bar. You should then get a message telling you that Cache has shut down.
